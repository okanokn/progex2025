5.3 include指令

コンパイラがソースコードを機械語に変換する前に，プリプロセッサがソースコードを編集する． このときの動作について説明する．

#include <ファイル名>があるとき，その場所にファイル名で指定したファイルの中身が挿入される． 既存のライブラリ（のヘッダファイル）など，システム上のファイルを指定する場合はファイル名を<と>で囲う． 自作のファイルを読み込む場合はファイルパスを"で囲う．

5.4 define指令（オブジェクトマクロ）

#define a bと書くと，これ以降のaはbに置換される（ただし，"で囲われた中身（文字列リテラル），変数名の一部は置換されない）． これをオブジェクトマクロと呼ぶ． プログラム中で使用する定数（プログラム実行中に書き換えることのない値）に名前をつけて可読性を上げたり，定数を変更する際の手間を省いたりするのに役立つ． たとえば，学生数分の配列を使う場合，下記のように書くことができる．

#define STUDENT_NUM 30
int main(void){
    int score[STUDENT_NUM]; //コンパイルの前に int score[30] に置き換えられる
    // ...
    for(int i = 0; i < STUDENT_NUM; i++) 
    // ...
}

オブジェクトの名前は，変数と区別するために大文字アルファベットで記述することが多い．

#define PI 3.14159

5.5 define指令（関数形式マクロ）

マクロを使って関数のようなものを定義できる． たとえば，2乗するマクロは以下の通り．

#define SQR(x) ((x) * (x))
int main(void){
    int a = SQR(2); // a = ((2) * (2)) に置換され a = 4 となる
}

関数形式（int sqr(int x){ return x*x; }）と比較すると，型を限定しない関数（ぽいもの）を定義できるという利点がある． 例えば，上のSQR()は，

int a = 2;
printf("%d\n", SQR(a));  // int型の変数を2乗
double b = 2.2;
printf("%lf\n", SQR(b)); // double型の変数を2乗

のように使うことができる．

同様に2変数関数も作ることができる．

#define MAX2(x,y) (x > y ? x : y)

ここで，定義部分は条件演算子と呼ばれ，(条件式 ? 条件式が真の場合の値 : 条件式が偽の場合の値)という書式である．

マクロには利点がある一方で，意図しない動作をする恐れがある点に注意が必要である． 例えばSQR(a++)は((a++) * (a++))に置換されることから，単にaの2乗を計算してからaをインクリメントするのではなく，インクリメントが2回行われる． もう一つの例として，

#define NIBAI(x) (x * 2)
int main(void){
    printf("%d\n", NIBAI(3+4)); // 3+4 * 2 と展開される
}

とすると，NIBAI(3+4)は3+4 * 2と展開される． よって，おそらく想定していた\(3+4 = 7\)の2倍である\(14\)とは異なり，値は\(3+8=11\)となる．

5.6 ifdef指令

もうひとつ#を使う指令として，ifdefを紹介する． これを使うと，特定の名前のマクロが定義済みか否かによってプログラムの実行を分岐させることができる．

#define マクロ名 値
// ...
#ifdef マクロ名
  「マクロ名」が定義済みの場合にのみ，この部分を実行
#endif

たとえば，デバッグ用にprintf文を仕込んだとき，それを実行するか否かをマクロの有無で切り替えることができる．

#define IS_DEBUG_MODE // デバッグモードでないときはコメントアウト
int main(void){
    #ifdef IS_DEBUG_MODE
        printf("デバッグ用メッセージ");
    #endif
}

5.7 文字

文字1字を表すにはchar型の変数を使う．1文字を表現するにはその文字を単引用符（'）で囲う． printfでchar型の文字を出力するには%cで書式指定する．

char moji = 'a'; // 文字「a」を変数mojiに代入
printf("%c\n", moji); // a を出力

charは1バイト分の整数（\(0\)–\(255\)または\(-128\)–\(127\)）を表現可能な型である． つまり，文字は，文字に対応する整数値で変数に保存される．

char moji = 'a';
printf("%d\n", moji); // 文字「a」に対応する数字 97 を出力

この整数値はASCII（アスキー）コードと呼ばれる．その一部を別表に示す．

マルチバイト文字

ここでは文字として半角英数と記号(!"#$%&'()*+,-./:;<=>?@[\]^_`{|}~)を考える． 日本語の文字は1バイトでは表現できず，1文字を2バイト以上で表現する． 文字と数字の対応は文字コードに依存する．

逆に，文字に対応する整数値を指定すれば，文字を表現することができる．

char moji = 97; // 整数値97を代入
printf("%c\n", moji); // 書式を%cで指定すると対応する文字「a」を出力

ASCIIコードを確認するとわかるが，連続するアルファベットは連続する数値が割り当てられている． また，Aとaは異なる文字なので，対応する数字も異なる．

5.8 文字列の基本

文字列は，文字の配列で表現する．つまりchar型の配列である． 文字を二重引用符（"）で囲うと文字列を表す． このように，ソースコード上の"で囲われた文字列の表現のことを文字列リテラルと呼ぶ．

char str[4] = "abc"; // 配列strを文字列リテラルで初期化

printfで書式指定を%sとすると，与えられた配列の文字列を出力する．

printf("%s\n", str); // 文字列に対応する書式指定は %s

ここで，配列strのサイズが4であることに注意する． これは，文字列は末尾に終端を意味する不可視の記号（NULL文字）を含むため． NULL文字は'\0'と表現する．

つまり，文字列の初期化

char str[4] = "abc";

を文字に分解して書き表すと

char str[4] = {'a', 'b', 'c', '\0'};

となり，両者の意味するところは同一である．

文字列の末尾にはNULL文字が必要

末尾にNULL文字が無い文字列はC言語の仕様に適さない．

char str[] = {'a', 'b', 'c'};
printf("%s\n", str);

とすると，abcに続いて変な文字が出力される．

5.9 配列としての文字列

文字列は配列であるから，普通の配列と同様に要素ごとに参照や代入が可能である．

char str[] = "abc"; // 配列を宣言と同時に初期化する場合はサイズを省略できる

// 文字列として出力
printf("%s", str); // abc
printf("\n");

// 1要素ずつ文字として出力
for(int i = 0; i < 4; i++){
    printf("%c, ", str[i]); // a, b, c
}
printf("\n");

// 1要素ずつ数値として出力
for(int i = 0; i < 4; i++){
    printf("%d, ", str[i]); // 97, 98, 99
}
printf("\n");

通常の配列と同様，初期化以外で一括代入はできない．

char str[4];
str = "abc"; // エラー

文字列の途中にNULL文字がある場合，最初のNULL文字までが文字列となる． たとえば"abc\0def"という文字列リテラルで初期化した配列を文字列として認識した結果はabcである．

char str[] = "abc\0def";
printf("%s\n", str); // abc
Note

文字列を初期化するときに，サイズ数より大きい文字列リテラルを代入するとNULL文字が入り切らずに切り取られて不正な文字列になるので注意．

char str[4] = "abcd"; // dの後のNULL文字が入りきらない
printf("%s\n", str);  // 意図しない出力

逆に配列のサイズが初期化文字列より大きい場合は，残りの配列要素が0，つまりNULL文字で埋められるので，問題は生じない．

char str[10] = "abcd"; // 5番目以降は 0 = '\0' で初期化
printf("%s\n", str);   // abcd

5.10 文字列の組

複数の文字列はchar型の2次元配列で扱うことができる．

char name[][11] = {"Alice", "Bob", "Charlie"}; // 長さ11(= 10字+NULL文字)の文字の組を格納する2次元配列

この2次元配列の要素は下記のようである．

name[0][0] = 'A', name[0][1] = 'l', name[0][2] = 'i',  name[0][3] = 'c', name[0][4] = 'e', name[0][5] = '\0', ..., name[0][10] = '\0',
name[1][0] = 'B', name[1][1] = 'o', ...,
name[2][0] = 'C', name[2][1] = 'h', ..., name[2][10] = '\0'

2次元配列の文字列を順に出力するプログラム（の一部）は以下の通り．

char name[][11] = {"Alice", "Bob", "Charlie"};
for(int i = 0; i< 3; i++){
    printf("%s\n", name[i]); // [i]が必要なことに注意
}

5.11 文字列の読み込み

scanf関数でユーザーの入力を文字列として読み込むことができる．

#include <stdio.h>
#define MAX_CHAR 10
int main(void){
    char str[MAX_CHAR+1];
    printf("文字列を入力してください:");
    scanf("%s", str); // %s = 文字列として解釈, &をつけないstr
    printf("%s と入力しましたね\n", str);
    return 0;
}

このとき，scanfに渡すstrに&が不要である点に注意． これは，配列名は配列先頭のアドレスを表すためであり，（配列ではない）変数にアドレス演算子&をつけたものと同種の情報を渡していることになる．

scanfはスペースを区切り文字として扱う

scanfはスペースで入力を区切るので，ユーザーがスペースを入力すると，その後の文字列はstrに格納されない． スペース（や改行）を含めて読み取るには，ファイル入出力に使うfgets関数を使うとよい．

fgets(str, sizeof(str), stdin); // 読み込み先の配列，配列サイズ，読み込み元 (stdinは標準入力)
printf("%s と入力しましたね\n", str);

5.12 stringライブラリの便利関数

string.hには文字列関係の便利な関数が定義されている． いくつか紹介する．

5.12.1 文字列の長さ

  • strlen(文字列): NULL文字を除く文字列の長さを返す
  • strcpy(配列名, 文字列): 配列名に文字列をコピーする
  • strcat(配列名, 文字列): 配列名内の文字列末尾に文字列を結合する
  • strcmp(文字列1, 文字列2): 文字列1と2を辞書式に大小比較し，文字列1が文字列2より小さい場合に負の整数，等しい場合に0，文字列1が文字列2より大きい場合に正の整数を返す
#include <stdio.h>
#include <string.h>
int main(void) {
  // 文字列の長さ
  char s1[] = "abc";
  unsigned int len = strlen(s1); // 文字列の長さを返す(NULL文字を除く)
  printf("%d\n", len);
  printf("%lu\n", sizeof(s1)); // sizeof(配列名)で配列のバイト数を返す. unsigined longなので%lu

  // 文字列のコピー
  char s2[10];
  strcpy(s2, s1);     // s2にs1をコピー
  printf("%s\n", s2); // abc

  // 文字列の結合
  char s3[10] = "xyz";
  strcat(s3, s1);     // s3の後にs1を連結 (s3は十分長くとっておく)
  printf("%s\n", s3); // xyzabc

  // 文字列の比較
  char s4[10] = "abcd";
  int cmp1 = strcmp(s1, s4);    // s1 < s2, 負
  int cmp2 = strcmp(s1, "abc"); // s1 == s2, 0
  printf("%d, %d\n", cmp1, cmp2);
  
  return 0;
}
