7.3 構造体

単一の数値や文字を保持するにはintやcharなど適当な型を設定した変数を用いて，一連のデータをまとめて扱うには，配列を使えばよかった． たとえば，あるクラスのテストの点数をまとめて管理するためにint型の配列を用いた． これに，テスト受験者の名前—つまり文字列データ—を紐づけて管理するにはどうすればよいか？

構造体 (structure) を使うと，異なるデータ型をひとまとめにできる．構造体は，その内部に包含するデータの型を宣言し，あたかも新たなデータ型のように使うことができる． つぎの例は，ある人物に関するデータをまとめた構造体の宣言である．

struct person{
  char name[128];  // 氏名
  float height;    // 身長
  float weight;    // 体重
  int age;         // 年齢
};

行頭にstructを付けると，構造体の宣言となる．personはその構造体の名前（構造体タグ (structure tag)）である． タグに続くブロック{}の中に，この構造体が含むデータ（メンバ (member)）のデータ型とその名前が定義されている．

構造体の宣言ブロック

構造体宣言ブロックの末尾には;が必要である．関数の定義と異なることに注意．

これは構造体personという型の宣言であって，personという名前の変数が作られるわけではない． 実際にデータを格納するには，“person型”の変数を宣言して，値を代入する．

#include <stdio.h>
#include <string.h> // 文字列をコピーする関数strcpyのために必要
struct person {
  char name[128];  // 名前
  float height;    // 身長
  float weight;    // 体重
  int age;         // 年齢
};

int main(void) {
  struct person ohtani;
  strcpy(ohtani.name, "Shohei Ohtani");
  ohtani.height = 1.93;
  ohtani.weight = 95.3;
  ohtani.age = 30;
  printf("%s (%d)\n", ohtani.name, ohtani.age);
  printf("H: %4.3f m, W: %4.2f kg\n", ohtani.height, ohtani.weight);
  return 0;
}

上のプログラムでは，3–8行目で構造体personを宣言している．この段階ではメモリ上にデータ領域は確保されていない．11行目で，“person型”の変数ohtaniが宣言され，ここでメモリ領域が確保される．大きさはchar[128] = 128バイト，float ✕ 2 = 8バイト，int ✕ 1 = 4バイトの計140バイトである 1． 12–15行目で，ohtaniのメンバに値が代入されており，そのデータは16, 17行目で参照されている．

.は構造体のメンバにアクセスする演算子である．

メンバにアクセスするための演算子

構造体変数struc_nameのメンバmem_nameにアクセスするには，struc_name.mem_nameと書く．

わざわざstrcpyを使う理由

メンバnameは（ポインタ変数ではなく）配列として宣言されているため，

ohtani.name = &"Shohei Ohtani"; // エラー

のように文字列リテラルのアドレスを代入して書き換えることはできない．また，既に宣言済みであって初期化ではないので

ohtani.name = "Shohei Ohtani"; // エラー

もコンパイルエラーとなる．

宣言と同時に初期化する場合には，配列の初期化と同様に{}の中に,区切りでメンバの値を指定する．上のプログラムのmain関数部分を以下のように書き換えた．

int main(void) {
  struct person ohtani = {"Shohei Ohtani", 1.93, 95.3, 30}; // 初期化
  printf("%s (%d)\n", ohtani.name, ohtani.age);
  printf("H: %4.3f m, W: %4.2f kg\n", ohtani.height, ohtani.weight);
  return 0;
}
メンバの初期化

配列の初期化と同様に，{}でメンバの一部のみ初期化した場合は，残りのメンバは0で初期化される．

7.4 構造体へのポインタ

構造体は複数のデータ種類を一つの変数にまとめることができるので，関数の引数をコンパクトに，わかりやすく表現することができる．たとえば，人物に関するデータ処理を行う関数は，名前，身長，体重などのデータを個別に渡すよりも，人物データをひとつの変数にまとめた方がわかりやすいし，保守性が高い．

#include <stdio.h>
#include <string.h>
struct person {
  char name[128];  // 氏名
  float height;    // 身長
  float weight;    // 体重
  int age;         // 年齢
};

void printBMI(struct person p) { // BMIを出力する関数
  printf("%s's BMI: %f\n", p.name, p.weight / p.height / p.height);
}

int main(void) {
  struct person ohtani = {"Shohei Ohtani", 1.93, 95.3, 30};
  printBMI(ohtani);
  return 0;
}

関数で構造体のデータを書き換えるには，構造体へのポインタを関数に渡して（参照渡し），呼び出し元における構造体を関数内で書き換える必要がある．

#include <stdio.h>
#include <string.h>
struct person {
  char name[128];  // 氏名
  float height;    // 身長
  float weight;    // 体重
  int age;         // 年齢
  double bmi;      // BMI
};

void calcBMI(struct person *p) { // BMIを書き込んで返す関数
  double b = (*p).weight / (*p).height / (*p).height;
  (*p).bmi = b;
}

int main(void) {
  struct person ohtani = {"Shohei Ohtani", 1.93, 95.3, 30};
  calcBMI(&ohtani);
  printf("BMI: %lf\n", ohtani.bmi);
}

ここで，(*p).bmiはp->bmiと書くこともできる．->をアロー演算子という． この表現を使うと，上の関数calcBMIはつぎのようにも書ける．

void calcBMI(struct person *p) { // BMIを書き込んで返す関数
  double b = p->weight / p->height / p->height;
  p->bmi = b;
}
アロー演算子

構造体へのポインタpのメンバmemを参照するには，(*p).memだけでなく

struct strTag *p; // 構造体 strTagへのポインタ
p->mem;           // = (*p).mem

と書くこともできる．

7.5 構造体の受け渡し

配列と異なり，構造体変数に別のオブジェクトを代入することができる．

struct person p1, ohtani = {"Shohei Ohtani", 1.93, 95.3, 30};
p1 = ohtani;  // 正常

したがって，構造体を返す関数を定義して，戻り値を呼び出し元で宣言した構造体変数に代入することもできる．つぎの例の関数createPersonは，必要な情報を受け取って，それらの情報を一つの構造体に加工して呼び出し元に返す関数である．

#include <stdio.h>
#include <string.h>
struct person {
  char name[128];  // 氏名
  float height;    // 身長
  float weight;    // 体重
  int age;         // 年齢
  float bmi;       // BMI
};

struct person createPerson(char name[], float h, float w, int a) {
  struct person p = {"", h, w, a};  // 構造体に詰めて返す
  strcpy(p.name, name);
  p.bmi = w / h / h;
  return p;
}

int main(void) {
  struct person p;
  p = createPerson("Shohei Ohtani", 1.93, 95.3, 30);
  // ... 構造 p を使った処理を書く
}

構造体の配列も，データ型の配列と同様に定義できる．

struct person player[10];
player[0] = createPerson("Shohei Ohtani", 1.93, 95.3, 30);
player[1] = createPerson("Aaron James Judge", 2.01, 127.9, 33);
// 後略

7.6 共用体

構造体は型の異なるデータをまとめて管理できるが，メンバを増やすと使用するメモリ領域が大きくなる．複数の型のデータを格納するが，同時に保持するのはひとつのメンバだけという場合には，共用体 (union) として宣言することでメモリ領域を節約できる．

たとえば，文字，整数，浮動小数点数のいずれかが入るデータ型keywordを定義したいとする．char, int, doubleの3種類をメンバとして持つ構造体は，

struct keyword{
  char c;   // 1バイト
  int i;    // 4バイト
  double d; // 8バイト
};

となり13バイト以上（データの切れ目を調整するパディングによって16バイト）が必要である．

しかし，メンバのうち複数を同時に保持することが無いのであれば，一番大きなdoubleに合わせてメモリ領域を確保しておき（つまり8バイト分），実際には格納しているデータ型に応じて読み出す領域を決めてあげればメモリを節約できる．これは共用体で実現できる．

union ukeyword{
  char c;   // 1バイト
  int i;    // 4バイト
  double d; // 8バイト
}; // 最大のdoubleに合わせて8バイト（宣言時に）確保される

共用体オブジェクトの参照には構造体と同じく.演算子を使う．

7.7.1 問題1

書籍の情報を保持する構造体を定義して，手元にある3冊の本の情報を構造体配列に保存せよ． 書籍1冊の情報には，以下を含むこと．

  • 書名
  • 著者名
  • 価格
  • ISBN
  • 発行年

この構造体を用いて，以下の動作を実現せよ．

  1. 本1冊の情報を出力する関数printBookInfoを作成し，それを用いて3冊全ての情報を出力する
  2. 発行年が古い順に配列を並び替える関数sortBookByYearを作成し，並び替えを実行する
  3. 再び3冊全ての情報を出力し，ソートが完了していることを確認する

プログラム名はq1.cとする．
