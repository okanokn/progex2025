6.3 ポインタの基本

ポインタとは，他の変数のメモリアドレスを格納する変数である． ポインタ変数ともいう．ポインタのデータサイズは，64ビットマシンなら64ビット（8バイト）である．

ポインタを宣言するには，データ型 *ポインタ名と書く． 例えば，int型のポインタは以下のように宣言する．

int a = 10;
int *p; // int変数へのポインタを宣言
p = &a; // 変数aのアドレスをpに格納

3行目で，&は変数のアドレスを抽出する演算子なので，pにはaのアドレスが格納される．データを確認してみよう．printfでアドレスを出力する書式指定は%pである．

printf("%p\n", &a); // 0x16b386c88 (変数aのアドレス)
printf("%p\n", p);  // 0x16b386c88 (9桁しかないのは上の桁が0で省略されているため)

2行目で，pの前には&がついていないことに注意．

ポインタに*をつけると，ポインタが指している変数（つまり，ポインタ変数に格納されているアドレスの変数）と同じ様に扱うことができる．*を間接演算子と呼ぶ．

int a, b;
int *p;  // int型へのポインタを宣言
a = 10;
p = &a;  // pにaのアドレスを格納
b = *p;  // b = a;と同じ
*p = 20; // a = 20と同じ
printf("%d\n", a);   // 20
printf("%d\n", *p);  // 20
*をつけてポイント先の変数にアクセス

このときの*は，ポインタ変数宣言時の*とは全く別物である． 変数宣言時の

int *p;

は，意味的には

int* p;

つまり，int型データのアドレスを格納するint*型変数を宣言していると考えるとよい． 実際，int型変数は4バイト，ポインタ変数は8バイト1の変数で別物である．

Tip

ポインタ変数pをint* p;と宣言することも可能だが，複数個のポインタ変数をまとめて宣言する場合に直観に反する動作をするので，書き方としてはint *p;を推奨する．

int* p1, p2, p3; // int *p1, p2, p3; と解釈される

は，p1はポインタ，p2とp3はint型変数となる．

Tip

通常の変数に対して*を使うことはできない．

int a = 10;
*a; // コンパイル時にエラー

6.4 関数の引数にポインタを使う

関数がreturnで返すことができるデータは1つだけであるが，ポインタを使うと2つ以上の値を得ることができる．

つぎの例は，2つの変数a, bを受け取って2種類の計算a+bとa-bを行う関数sum_diffを定義している．

#include <stdio.h>
void sum_diff(int a, int b, int *add, int *diff) {
  *add = a + b;  // ポインタaddの指すアドレスに値を書き込む
  *diff = a - b; // ポインタdiffの指すアドレスに値を書き込む
}

int main(void) {
  int ad, df; // 戻り値を格納するint型変数
  sum_diff(10, 100, &ad, &df); // 戻り値用の *アドレス* を関数に渡す
  printf("a-b: %d, a-b: %d\n", ad, df);
  return 0;
}

関数の引数int *add, int *diffは計算結果の格納用で，int型のポインタとして受け取っている． 呼び出し元のmain関数では，戻り値の書き込み先のアドレス&ad, &dfを渡している（9行目）．関数sum_diff内では，*演算子を使ってアドレスで参照している先，つまりmainで宣言された変数にアクセスして計算結果を書き込んでいる．

ここで，関数の引数をポインタとせず，通常のint変数で渡すと，エラーにはならないが意図した動作は実現できない． これは，ポインタではない通常の変数を関数に渡すときには，関数内の変数に値がコピーされ，呼び出し元とは別の領域にデータが格納されるためである．これを値渡しという． これに対して，呼び出し元で用意したデータ格納先のアドレスを渡すと，関数内から呼び出し元の変数にアクセスすることができる．これを参照渡しと呼ぶ．

値渡しと参照渡しの動作の違いを確認するため，渡された2つの数字を入れ替える関数の実装例を見てみよう．

#include <stdio.h>
void swap1(int a, int b) {// aとbを入れ替える（値渡し）
  int tmp = a;
  a = b;
  b = tmp;
}

void swap2(int *a, int *b) {// aとbを入れ替える（参照渡し）
  int tmp = *a;
  *a = *b;
  *b = tmp;
}

int main(void) {
  int a = 1;
  int b = 10;
  swap1(a, b);  // 値渡し
  printf("swap 1: a = %d, b = %d\n", a, b); // 入れ替わらない

  a = 1;
  b = 10;
  swap2(&a, &b); // 参照渡し
  printf("swap 2: a = %d, b = %d\n", a, b); // 入れ替わる

  return 0;
}

6.5 ポインタと配列

6.5.1 ポインタで配列を走査

配列は，メモリ上に連番で格納領域を確保するので，連続したアドレスが並ぶことになる．

int a[3] = {0, 1, 2};
int *p = &a[0];

上の例では，ポインタpに配列の先頭a[0]のアドレスが格納される．

ここでpの値を1だけ増加させる．すると，pの値はint型データ1つ分のアドレス（つまり4バイト分）が加算され，a[1]のアドレスと一致する．

p++;                // pをインクリメント
printf("%d\n", *p); // 1 (=a[1])

同様に，aがdouble型の配列でpがdouble型へのポインタであれば，pに1を加えるとdouble型データ1つ分のアドレス（つまり8バイト分）が加算される．

#include <stdio.h>
int main(void) {
  double a[] = {0.0, 1.0, 2.0};
  double *p, *q;

  p = &a[0];
  q = p + 1;
  printf("%lf\n", *q);      // 1.0 (=a[1])
  printf("%p, %p\n", p, q); // 0x16cf4ec70, 0x16cf4ec78
  return 0;
}

6.5.2 配列名とアドレス

第05回で触れたが，配列名単独は，配列の先頭を表すアドレスを意味する．

int a[] = {0, 1, 2};
int *p;
p = a; // p = &a[0]と同じ

また，配列要素のアドレスは配列名 + インデックス番号と書くことができる． これは，配列名がその配列の先頭へのポインタであることを考えると納得できるであろう．

a;     // = &a[0]
a + 1; // = &a[1]
a + 2; // = &a[2]

これを利用して，配列aのi番目要素を*(a + i)のように参照することができる．

*(a + 1) = 0; // a[1]に 0 を代入

まとめると，配列aのインデックス1番目を参照する方法として，以下の3種類は同じ意味である．

int a[] = {0, 1, 2};
int *p = &a[1]; // a[1]へのポインタ
a[1];     // a[1]の値
*p;       // a[1]の値
*(a + 1); // a[1]の値

6.5.3 配列を引数とする関数

配列を引数とする関数は次のように定義する．

// 配列とその長さを受け取って，すべての要素を出力する関数
void print_array(int a[], int n){
  for(int i = 0; i < n; i++){
    printf(" %d", a[i]);
  }
  printf("\n");
}

これは，配列先頭へのポインタを渡す関数としても定義できる．

// 上の関数の引数の表現をポインタに変えたもの
void print_array(int *a, int n){
  for(int i = 0; i < n; i++){
    printf(" %d", a[i]);
  }
  printf("\n");
}

ここで，関数の中身（3–6行目）は全く同一であることに注意．つまり，ポインタpに対してp[i]との表記も可能であり，これは*(p + i)を表す．

Tip

関数の引数を配列形式int a[]とするかポインタ形式int *aとするかは，読ませたい意図によって使い分けることが多いようである．つまり，配列としての意味を強調したいのか，ポインタとして読ませたいかに依る． いずれの場合でも，配列のサイズを超えてアクセスすることがないように，配列のサイズを併せて関数に渡す．

配列名とポインタが全く同一のように扱えるか，というとそうではない． たとえば，配列aに別の配列のアドレスを代入することはできない．

int a[] = {0, 1, 2}
int *p = a;
int b[] = {3, 4, 5};
p = b; // OK
a = b; // コンパイル時にエラー
