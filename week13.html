13.3 ソートの基礎

数値，文字，日付など順序が定義できるデータを，昇順や降順に並び替えることをソート（sort）するという．ソートすること自体が目的の場合や，さまざまなアルゴリズムで手順の一部として登場する．

ソート方法の評価ポイントは，代表的に以下の3点である．

  • 時間計算量: データ数に対してどの程度の手順でソートが完了するか．元のデータの並びによって計算量が変わる場合は最悪，平均，最良の計算量をそれぞれ考える．
  • 必要メモリ量: 並び替えたいデータの配列に加えて，別途必要となる記憶域がある場合，どの程度か．
  • 安定性: 同一の値を持つデータが複数ある場合，ソート後に元の順序が保存されるかどうか．

以下では，配列で与えられた整数データを昇順に並び替えることを考える． データを数列\(a_i\), \(i=0,1, \dots, n-1\)で表記する．

13.4 挿入ソート

直観的にわかりやすい挿入ソートを紹介する．これは，配列の左側をソート済み，右側を未整列として，未整列の先頭を，ソート済みの適当な箇所に挿入することを繰り返すものである．

  1. ソート済みの末尾インデックスを表す\(i\)を\(1\)に設定する（配列の最初\(a_0\)はソート済みと見なす）．
  2. \(i \geq n\)ならばすべて並べ終わったので終了．そうでなければ，次の手順に進む．
  3. \(a_i\)を挿入する場所を求める．\(j\leftarrow i\)として以下の手順を繰り返す．
    • \(j \geq 1\)かつ\(a_j < a_{j-1}\)の間，\(a_j\)と\(a_{j-1}\)を交換し，\(j \leftarrow j - 1\) とする
    • それ以外の場合，\(a_j\)の挿入が完了したので次の手順に進む．
  4. \(i\leftarrow i+1\)として手順2に戻る．

ソート部分の関数の実装例は以下のとおり．

// 値を入れ替える関数
void swap(int* a, int* b) {
  int tmp = *a;
  *a = *b;
  *b = tmp;
}
// 挿入ソート
void insertionSort(int a[], int n) {
  for (int i = 1; i < n; i++) {
    int j = i;
    while (j >= 1 && a[j] < a[j - 1]) {
      swap(&a[j], &a[j - 1]);
      j--;
    }
  }
}

挿入ソートの最悪計算量は\(O(n^2)\)である．

13.5 マージソート

整列対象のデータ数が多いと，比較対象が多いので挿入ソートでは計算量がかさむ．そこで，ソートしたい配列を分割して部分配列ごとにソートを行い，それらを併合（マージ（merge））することで全体のソートを達成する方法がある．マージソートという．手順は以下のとおり．

  1. 配列のサイズが1以下であれば何もしない（ソート済み）．それ以外の場合，配列を2分割し，各部分裂に対して再帰的に同じ手順を適用する．
  2. ソート済みの左部分列と右部分列の先頭（それぞれの最小値）を比べ，小さい方から「ソート済み配列」に順に書き込む．
  3. どちらかの部分列が尽きたら，他方の残りをすべて書き込む．

手順1の再帰的な分割は以下のように実装できる．

// a[p], ..., a[p+len-1]をマージソート
void mergeSort(int a[], int b[], int p, int len) {
  if (len > 1) {
    int m = len / 2;
    mergeSort(a, b, p, m);
    mergeSort(a, b, p + m, len - m);
    merge(a, b, p, len); // マージ
  }
}

配列aは与えられた配列で，配列bは同じ長さのソート結果を書き込むための配列である． 6行目の関数mergeでは左部分列a[p], …, a[p+len/2-1]と，右部分列a[p+len/2], …, a[p+len-1]のマージを行っている．

void merge(int a[], int b[], int p, int len) {
  int i = p;            // 左部分列のindex: [p, p + len/2 - 1]
  int j = p + len / 2;  // 右部分列のindex: [p + len/2, p + len-1]
  // マージしてb[k]に書き込む
  for (int k = p; k < p + len; k++) {
    if (j == p + len || (i < p + len / 2 && a[i] <= a[j])) {
      b[k] = a[i];
      i++;
    } else {
      b[k] = a[j];
      j++;
    }
  }
  // bの該当部分をaにコピー
  for (int k = p; k < p + len; k++) {
    a[k] = b[k];
  }
}

5行目のkに関するfor文は，b[k]の要素を順に決定していくループである． 6行目のifは，

  • 右部分列が空
  • 左部分列が空でない かつ 左先頭が右先頭より小さい

のどちらかが真であれば，左部分列の要素a[i]を，そうでなければ右部分列の要素a[j]をb[k]に書き込む．

マージソートの実装例 
#include <stdio.h>

void merge(int a[], int b[], int p, int len) {
  int i = p;            // 左部分列のindex [p, p + len/2 - 1]
  int j = p + len / 2;  // 右部分列のindex [p + len/2, p + len-1]
  // マージしてb[k]に書き込む
  for (int k = p; k < p + len; k++) {
    if (j == p + len || (i < p + len / 2 && a[i] <= a[j])) {
      b[k] = a[i];
      i++;
    } else {
      b[k] = a[j];
      j++;
    }
  }
  // bの該当部分をaにコピー
  for (int k = p; k < p + len; k++) {
    a[k] = b[k];
  }
}

void mergeSort(int a[], int b[], int p, int len) {
  if (len > 1) {
    int m = len / 2;
    mergeSort(a, b, p, m);
    mergeSort(a, b, p + m, len - m);
    merge(a, b, p, len);
   }
}

void printArray(int a[], int n) {
  for (int i = 0; i < n; i++) {
    printf("%d ", a[i]);
  }
  printf("\n");
}

int main(void) {
  int data[] = {7, 2, 9, 4, 3, 6, 1, 8, 5};
  int n = sizeof(data) / sizeof(data[0]);
  int tmp[n];
  printf("ソート前: ");
  printArray(data, n);

  mergeSort(data, tmp, 0, n);
  
  printf("ソート後: ");
  printArray(data, n);
  return 0;
}

長さ\(l\)の部分列同士をマージする際に\(l\)回の比較が行われる． これが全体で\(n/(2l)\)回行われるので，長さ\(l\)の部分列のマージにかかる計算量は\(O(n)\)である． 分割により生じる部分列は\(\log_2 n\)本であるから，アルゴリズム全体で最悪・平均時間計算量は\(O(n\log n)\)である．

比較によってソートするアルゴリズムとしては最良の計算量\(O(n\log n)\)を達成できるが，別途\(O(n)\)のメモリが必要である．

13.6 クイックソート

マージソートと同じく，小さな部分列に分割して処理する方法としてクイックソート（quicksort）がある． これは分割の際に工夫がある．

  1. \(a_i\ (i=0,\dots,n-1)\)のある値を基準値（ピボット）に設定する．
  2. ピボット以下の\(a_i\)を左に，ピボットより大きい\(a_i\)を右に移動する．
  3. 左右の部分列について再帰的に同じ手順を適用する．部分列の長さが1以下ならばソート済みで終了．

主要部分の実装例は以下のとおり．

void partition(int a[], int l, int r, int* l_end, int* r_start) {
  int i = l;
  int j = r;
  int pivot = a[l];
  while (i <= j) {
    while (a[i] < pivot) i++; // pivot以上を発見するまでiを右に
    while (a[j] > pivot) j--; // pivot以下を発見するまでjを左に
    if (i <= j) {
      swap(&a[i], &a[j]);
      i++;
      j--;
    }
  }
  *l_end = j;    // 分割後左部分列の最後尾
  *r_start = i   // 分割後右部分列の先頭
}

void quickSort(int a[], int l, int r) {
  if (l < r) {
    int m1, m2;
    partition(a, l, r, &m1, &m2);
    quickSort(a, l, m1);
    quickSort(a, m2, r);
  }
}

この例ではa[l]つまり（いま扱っている部分配列a[l], …, a[r]の）先頭の値をピボットに設定している． 左（右）の端からピボット以上（以下）の値を探してiをインクリメント（jをデクリメント）し（6, 7行目），見つかったa[i]とa[j]を交換する（8, 9行目）． （ただしiがjより右にある場合は交換しない．）

左端から出発したiが右端から出発したjと重なるか追い越したら探索を終了する（5行目）．

最後に，探索終了時のiとjを新たな部分列の切れ目としてポインタに書き込んで返す（14, 15行目）．

クイックソートの時間計算量は，元の配列の並びとピボットに選択する値に依るが，全並びパターン\(n!\)が等確率で出現するとすると，平均時間計算量は\(O(n\log n)\)である． 最悪ケースでは\(O(n^2)\)となる．

必要なメモリ量は，再帰的な分割を行う過程でスタックとして\(O(\log n)\)が必要である．

13.7 バケットソート

比較に基づくソートでは\(O(n\log n)\)が最良であるが，ある種の仮定が成り立つ場合に更に高速なアルゴリズムがある．

仮定

ソート対象のデータは\(m\)パターンである．ここで\(m\)は正の整数である． 簡単のため，以下ではソート対象のデータは\(0\)以上\(m\)未満の整数とする．

このとき，\(0\)から\(m-1\)までのラベルをつけた\(m\)個のバケツを用意しておき，整列対象の配列がどの値を何個含むかをカウントする． 最後に小さいラベルのバケツから順に取り出せばソートが完了する．これをバケットソート（bucket sort）という．

手順は以下のとおり． 1. 数列\(b_0, b_1, \dots, b_{m-1}\)を用意する． 1. 各\(a_i\)について，その値に対応する\(b_*\)の値をインクリメントする．つまり，\(b_{a_i}\leftarrow b_{a_i} + 1\)． 1. \(b_0\)から順に，\(b_j\)の値だけ\(j\)を並べる．

値の範囲が\(0--9\)の10種類の場合の実装例を以下に示す．

#define M 10  // 値の範囲（0〜9）

void bucketSort(int a[], int n) {
  int bucket[M] = {0};
  // 各値の出現回数をカウント
  for (int i = 0; i < n; i++) {
    bucket[a[i]]++;
  }
  // カウントに基づいてソートされた配列を構築
  int offset = 0;
  for (int i = 0; i < M; i++) {
    for (int j = 0; j < bucket[i]; j++) {
      a[offset++] = i;
    }
  }
}

a[offset++] = iは，a[offset]にiを代入した後，offsetをインクリメントする動作を表す．

バケットソートの実装例 
#include <stdio.h>
#define M 10  // 値の範囲（0〜9）

void bucketSort(int a[], int n) {
  int count[M] = {0};
  // 各値の出現回数をカウント
  for (int i = 0; i < n; i++) {
    count[a[i]]++;
  }
  // カウントに基づいてソートされた配列を構築
  int offset = 0;
  for (int i = 0; i < M; i++) {
    for (int j = 0; j < count[i]; j++) {
      a[offset++] = i;
    }
  }
}

int main(void) {
  int data[] = {7, 3, 5, 2, 0, 9, 0};
  int n = sizeof(data) / sizeof(data[0]);
  printf("ソート前: ");
  printArray(data, n);
  bucketSort(data, n);
  printf("ソート後: ");
  printArray(data, n);
  return 0;
}

バケットソートの時間計算量は\(O(n+m)\)である． 整列対象に関して仮定が必要であるが，\(m\propto n\)であれば\(O(n)\)でソートを行うことができる． \(m\)が非常に大きい場合，メモリを確保できない場合がある．その場合は，桁ごとにバケットソートを行う方法がある．

13.8 演習

13.8.1 問題1

クイックソートの実装例で，関数partitionにおけるピボットの選択方法がいくつか提案されている． ピボットが整列対象の最小値や最大値の場合に最悪ケースで，中央値が最良である（なぜか？）． ソート前にその値を知ることはできないが，配列の中央付近の値をピボットとして選択する方法がある． この方法でクイックソートを実装し，正しくソートできることを確認せよ．

13.8.2 問題2

マージソートのマージ処理について，上とは別の実装を考える． ソート済み部分列a[p], …, a[q]とa[q+1], …, a[q+r]をマージしてソートされたa[p], …, a[q]を作成する関数mergeを下を参考に作成せよ．

void merge(int a[], int p, int q, int r) {
    int n1 = q - p + 1;
    int n2 = r - q;
    int L[n1], R[n2];
    int i; // 左部分列のインデックス
    int j; // 右部分列のインデックス
    for (i = 0; i < n1; i++) L[i] = a[p + i];
    for (j = 0; j < n2; j++) R[j] = a[q + 1 + j];

    // LとRをマージ
    i = 0;
    j = 0;
    int k = p; // 書き込み対象のインデックス
    while (i < n1 && j < n2) {
        // ここを埋める
    }
    while (i < n1) a[k++] = L[i++]; // 左部分列に残りがある場合の処理
    while (j < n2) a[k++] = R[j++]; // 右部分列に残りがある場合の処理
}
